Brief

The brief for this project is to create a database to attach to our website. We have previously built the foundations of a website and to develop the complexity we have to be able to manage running a database and connect it with the website. Alongside other heads of pride groups in wellington we have started an orgnaisation called Wellington queer students association to help connect and share ideas with queer youth in the greater Wellington region. Since I am head of social media to advertise and help organise I have decided to create a website for WQSA.

Summary of code in all tables

Member table

create table member(
            member_id integer primary key autoincrement not null,
            name text not null,
            email text not null unique,
            password text not null,
            authorisation integer not null
        );

News

create table news(
            news_id integer primary key autoincrement not null,
            title text not null unique,
            subtitle text not null unique,
            content text not null unique,
            newsdate date not null,
            member_id integer not null,
            foreign key(member_id) references member(member_id)
        );

Comments

create table comments(
            comments_id integer primary key autoincrement not null,
            comments_content text not null unique,
            comments_date date not null,
            member_id integer not null,
            news_id integer not null,
            foreign key(member_id) references member(member_id),
            foreign key(news_id) references news(news_id)
        );

Events

create table events(
            events_id integer primary key autoincrement not null,
            events_title text not null unique,
            events_content text not null unique,
            events_date date not null,
            member_id integer not null,
            foreign key(member_id) references member(member_id)
        );

Sprint Two

Plan

This sprint I need to create a new page to keep up to date with news that is going on to do with WQSA. The news page will be a separate database but will connect to the members database through the member id. This is important because I want each news item to print out the corresponding member name connected with each member id. This means that people can see who made the news article. The news page also needs to have a date and be in chronological order to make sure you can see the most recent news at the top of the page. The database needs to include the title, subtitle, content, name of the person who posted it and the date it was published.

Accessability

I have addressed website accessibility in this project because the majority of my font sizes are bigger size. There aren’t huge blocks of text that people could find hard to read and understand. If there are bigger parts of text they are in a bigger font so it isn’t much writing it’s just bigger. This font is also easier for people to read. I also have a navigation bar which is typical in websites so it will be easy for users to navigate around the website because none of it is in tech language. It is all in colours that people can read, either in black or white depending on which colour the box is. The navigation bar is clear the page links are on one side and the sign-in and log in are on the right-hand side which they typically are in websites so it is familiar with the users.

I have addressed the aesthetics and relevant implications throughout this project by making the website look aesthetically pleasing for the type of people that will be going onto this website. Since it is for a Wellington queer youth association I made sure to style my website with lots of different colours, especially on the home page because each box has a separate colour and it goes down according to the rainbow.
