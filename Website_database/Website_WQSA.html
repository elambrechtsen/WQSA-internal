Brief

The brief for this project is to create a database to attach to our website. We have previously built the foundations of a website and to develop the complexity we have to be able to manage running a database and connect it with the website. Alongside other heads of pride groups in wellington we have started an orgnaisation called Wellington queer students association to help connect and share ideas with queer youth in the greater Wellington region. Since I am head of social media to advertise and help organise I have decided to create a website for WQSA.

Summary of code in all tables

Member table

create table member(
            member_id integer primary key autoincrement not null,
            name text not null,
            email text not null unique,
            password text not null,
            authorisation integer not null
        );

News

create table news(
            news_id integer primary key autoincrement not null,
            title text not null unique,
            subtitle text not null unique,
            content text not null unique,
            newsdate date not null,
            member_id integer not null,
            foreign key(member_id) references member(member_id)
        );

Comments

create table comments(
            comments_id integer primary key autoincrement not null,
            comments_content text not null unique,
            comments_date date not null,
            member_id integer not null,
            news_id integer not null,
            foreign key(member_id) references member(member_id),
            foreign key(news_id) references news(news_id)
        );

Events

create table events(
            events_id integer primary key autoincrement not null,
            events_title text not null unique,
            events_content text not null unique,
            events_date date not null,
            member_id integer not null,
            foreign key(member_id) references member(member_id)
        );

Email


            create table email(
    email_address text not null unique,
    email_date date not null

Sprint one

Plan

For the first sprint of my database, I need to make a feature where you can become a website member and have a news page to print out important news relating to the website. I will first need to make the databases for the members and the news table. Then I will make some queries for each table and then I will move onto focusing on the front end for both the member table and the news table. This means you can access certain parts of the website but not others. This would include being able to write comments on the news page but not having access to add news or delete them. I will confirm the details with them first. They will also need to enter their credentials and password for the site. Then I need to add the login feature for the members so that it checks their authorisation so they can have access to what they should. I need to create a new page to keep up to date with news that is going on to do with WQSA. The news page will be a separate database but will connect to the members database through the member id. This is important because I want each news item to print out the corresponding member name connected with each member id. This means that people can see who made the news article. The news page also needs to have a date and be in chronological order to make sure you can see the most recent news at the top of the page. The database needs to include the title, subtitle, content, name of the person who posted it and the date it was published.
